# Coding Katas

Katas are a great way to train and flex your programming muscles. No matter if in a group of fellow hackers or alone, katas are a fun way to pass time!

**How does one approach katas?**

1. [Search engine of your liking] is your friend. No seriously, [search engine of your liking] is your friend. 

2. And [stack overflow](https://stackoverflow.com/), a peer-to-peer-driven forum for any code related question you can imagine, like [this one](https://stackoverflow.com/questions/5767325/how-can-i-remove-a-specific-item-from-an-array) or [this](https://stackoverflow.com/questions/1789945/how-to-check-whether-a-string-contains-a-substring-in-javascript)

3. Another tool we recommend is [Rubber Duck Debugging](https://rubberduckdebugging.com/). Understanding what your code is supposed to do is half the deal. Before writing code it is also helpfull to write [Pseudo Code]() and write comments that explain the rough idea line by line of what needs to happen in order to calculate a value, print a certain variable or perform an algorithm.

4. The other half is figuring out the correct way of telling the computer to obey by your commands. In the beginning it is hard to tell all the special charaters like [], {} or ``, syntaxtes and keywords apart. Going through your code line by line, even character by character if nessecary is a common practice for cases where you have connsecutive sleepless nights over a problem you can not solve only to realize you typed an O instead of a 0 (zero). This is something devs with any years of experience encounter, so don't beat yourself up if you cant figure out why your code has red scribbles all over the place and everything is in another color then before.

5. Speaking of colors, here are some awesome themes (they really do help in telling apart different parts of your code like variables, keyword, keys or strings) and VS Code hacks to make your developer live more fun
- [Quokka.js](https://marketplace.visualstudio.com/items?itemName=WallabyJs.quokka-vscode)
A cool extension that logs the value of a variable right next to it. No `node index.jadi-jada` nessecary!
- [SynthWave '84](https://marketplace.visualstudio.com/items?itemName=RobbOwen.synthwave-vscode)
No one really recommends the glow effect, though it is fun... for a minute or two
- Format your code regularly

Feel free to share your valuable findings, we will add them to list #sharingIsCaring #camelCaseVariablesForTheWin

The katas listed here are meant as a rough guideline, feel free to branch of and do different one

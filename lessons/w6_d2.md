## props vs data

### props

[Documentation](https://vuejs.org/v2/guide/components-props.html)

- passed to child by a parent
- can be a static value or dynamic data
- child has "read only" abilities towards the data

Props are passed like this

```html
<Header text="hello" v-bind:my-text="some more text"></Header>
```

and the child component accords for the props like this

```html
<script lang="ts">

export default Vue.extend({
  props: {
    text: {
      type: String
    },
    myText: {
      type: String
    }
  }
});
</script>
```

### data

[Documentation](https://vuejs.org/v2/guide/components.html#data-Must-Be-a-Function)

- data store for our applications and websites
- can be seen as "lets and consts" inside our code which can be used to be displayed or being passed as props to children
- can be updated, but for now we are going to use it as a static data source

```html
<script lang="ts">
import Vue from "vue";
import Header from "@/components/Header.vue";
import Hero from "@/components/Hero.vue";
import Trial from "@/components/Trial.vue";
import Testimonial from "@/components/Testimonial.vue";
import FeatureGrid from "@/components/FeatureGrid.vue";

export default Vue.extend({
  data() {
    return {
      toDoItems: [
          "Water plants",
          "Clean Windows",
          "Pickle Vegetables"
      ]
    };
  }
});
</script>
```

## Directives

- [Directives Overview](https://flaviocopes.com/vue-directives/#:~:text=Directives%2C%20are%20basically%20like%20HTML,the%20Vue%20directives%20in%20detail)
- [Directives Documentation]()

Vue.js uses a templating language that's a superset of HTML. We can use **interpolation** `<h1>{{title}}</h1>` and **directives** to extend our HTML powers. Directives are being used to embed (dynamic) data into our vue templates. It involves a keyword, usually prefixed with a v- e.g. v-on, v-for.

Before we introduce directives, let's recap a concept we learned yesterday and combine it with a new concept: **props** and **data**

We learned that **prop** are used to feed a child component with data. The child has "read only" abilities towards the props. **data** on the other hand 


### Example 1 v-html & v-on:click

Using the **v-html** directive we can render dynamic HTML that was e.g. sent by the server. It's a handy tool when creating placeholder elements.

The **v-on** directive can put an event listener on to every element.

```html
<template>
  <div>
    <div v-html="myTitle" v-on:click="logSomething"></div>
  </div>
</template>

<script lang="ts">
import Vue from "vue";

export default Vue.extend({
  data() {
    return {
      myTitle: "<h1>Welcome to my Site!</h1>"
    };
  },
  methods: {
    logSomething() {
      console.log("Hello");
    }
  }
});
</script>
```

### Example 2: v-text

The content is set using the **v-text** directive

```html
<template>
  <div>
    <h1 v-text="myTitle"></h1>
  </div>
</template>

<script lang="ts">
import Vue from "vue";

export default Vue.extend({
  data() {
    return {
      myTitle: "Welcome to my Site!"
    };
  }
});
</script>

```

### Example 3: v-if
**v-if** conditionally displays an element if the condition is met.

```html
<template>
  <div>
    <h1 v-if="display">Am I visible?</h1>
  </div>
</template>

<script lang="ts">
import Vue from "vue";

export default Vue.extend({
  data() {
    return {
      display: false
    };
  }
});
</script>
```


### Example 4: v-for & v-bind

[Documentation](https://vuejs.org/v2/guide/list.html)

**v-for** gives you the power to iterate over an object or an array and use its data to dynamically feed your HTML templates without having to write multiple markups for the similar elements.



```html
<template>
  <div>
    <section class="feature-grid">
      <h2>Do more with Dropbox Business</h2>
      <div class="feature-box">

        <div
          v-for="(feature, index) in features"
          v-bind:key="index"
          class="feature"
        >
          <img v-bind:src="feature.imageSource" alt="" />
          <h4>{{ feature.title }}</h4>
          <p>
            {{ feature.text }}
          </p>
        </div>

      </div>
    </section>
  </div>
</template>

<script lang="ts">
import Vue from "vue";

import feature1 from "@/assets/img/features_1.png";
import feature6 from "@/assets/img/features_2.png";

export default Vue.extend({
  data() {
    return {
      features: [
        {
          title: "Do less switching between tools",
          imageSource: feature1,
          text:
            "Tackle your everyday tasks with one tool. Create, edit, and share cloud content from Google Docs, Sheets, and Slides, Microsoft Office files, and Dropbox Paper right from Dropbox."
        },
        {
          title: "Get peace of mind",
          imageSource: feature2,
          text:
            "A secure, distributed infrastructure—plus admin tools for control and visibility—keep your company’s data safe on Dropbox.",
        }
      ]
    };
  },
});
</script>



```

## Notes and Hints for smoother developing

If you import images such as `.png` or `.svg`, make sure that your `shims-vue.d.ts` looks like this: (you can search for files using Cmd+P/Ctrl+P)
```js
declare module '*.vue' {
  import Vue from 'vue'
  export default Vue
}

declare module "*.svg" {
  const content: any;
  export default content;
}

declare module "*.png" {
  const content: any;
  export default content;
}
```
